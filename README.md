# Сервис аутентификации и авторизации

Сервис предоставляет API для управления пользователями, аутентификацией. Реализован на FastAPI с использованием PostgreSQL для хранения данных и Redis для кэширования и управления сессиями.

## Функциональность

1. **Регистрация пользователя**
   - POST `/api/v1/auth/register`
   - Валидация данных (логин, email, пароль)
   - Проверка уникальности логина/email
   - Хеширование пароля
   - Создание пользователя в БД

2. **Вход в аккаунт**
   - POST `/api/v1/auth/login`
   - Проверка логина и пароля
   - Генерация JWT access токена и refresh токена
   - Сохранение refresh токена в БД
   - Запись в историю входов (IP, user-agent, время)

3. **Обновление access токена**
   - POST `/api/v1/auth/refresh`
   - Валидация refresh токена
   - Генерация новой пары токенов
   - Инвалидация старого refresh токена

4. **Выход из аккаунта**
   - POST `/api/v1/auth/logout`
   - Инвалидация refresh токена
   - Очистка токенов пользователя

5. **Изменение данных пользователя**
   - PATCH `/api/v1/users/me` (изменение логина)
   - PATCH `/api/v1/users/me/password` (изменение пароля)
   - Проверка текущего пароля при смене
   - Валидация новых данных

6. **История входов**
   - GET `/api/v1/users/me/login-history`
   - Получение истории входов текущего пользователя
   - Пагинация результатов

## Технологии

- **FastAPI** – веб-фреймворк для построения API
- **PostgreSQL** – реляционная база данных
- **Redis** – кэш и хранилище сессий
- **SQLAlchemy** – ORM
- **Pydantic** – валидация данных
- **JWT** – JSON Web Tokens для аутентификации
- **Docker** – контейнеризация
- **Uvicorn** – ASGI-сервер

## Запуск проекта

### Предварительные требования
- Установленный Docker и Docker Compose
- Python 3.11+ (для локального запуска без Docker)

### Запуск с помощью Docker Compose

1. Клонируйте репозиторий:
   ```bash
   git clone <repository-url>
   cd Auth_sprint_1-main
   ```

2. Создайте файл `.env` на основе `.env.example`:
   ```bash
   cp .env.example .env
   ```
   При необходимости отредактируйте переменные окружения.

3. Запустите сервисы:
   ```bash
   docker-compose up --build
   ```

4. Сервис будет доступен по адресу: [http://localhost:8000](http://localhost:8000)

   Документация API (Swagger UI): [http://localhost:8000/api/openapi](http://localhost:8000/api/openapi)

### Локальный запуск (без Docker)

1. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

2. Запустите PostgreSQL и Redis (например, через Docker или локально).

3. Запустите сервер:
   ```bash
   python src/main.py
   ```
